## 2. O Mecanismo de Operação: "Thermodynamic Censorship"

A resolução de problemas NP sem força bruta é alcançada através da **Censura Termodinâmica**.

### Fase de Derretimento (Alta Temperatura)

![Phase](https://img.shields.io/badge/Fase-Derretimento-orange)

Inicialmente, em $T_{high}$, o sistema apresenta entropia máxima. Os nós existem em superposição de estados (Espuma Quântica), onde todas as rotas possíveis coexistem com probabilidades equiparáveis.

### O Processo de Computação (Resfriamento/Expansão)

![Process](https://img.shields.io/badge/Processo-Resfriamento-blue)

A redução gradual de $T$ (análogo ao resfriamento cósmico) altera a distribuição de probabilidade dos estados conforme o **Fator de Boltzmann**:

$$P(S) \propto e^{-\frac{E(S)}{k_B T}}$$

* **Supressão Exponencial:** Caminhos de alta energia (soluções ineficientes) têm sua probabilidade suprimida exponencialmente.
* **Seleção Natural de Dados:** O sistema não verifica caminhos incorretos; estes simplesmente deixam de ser acessíveis no espaço de fases por instabilidade termodinâmica.

### Fase de Cristalização (Baixa Temperatura)

![Phase](https://img.shields.io/badge/Fase-Cristalização-brightgreen)

Quando $T \to 0$, ocorre uma **Transição de Fase**. O sistema "congela" no estado fundamental (*Ground State*). A geometria resultante representa a solução do problema computacional.

## 3. Vantagem sobre Turing (O Salto O(1))

Na computação clássica, a minimização de funções complexas exige iteração sequencial:
`Passo 1 -> Passo 2 -> ... -> Passo 10^50` (Inviável para problemas NP-Hard).

**No Processador Entrópico:**

1. O tempo de resolução é determinado pelo **Tempo de Relaxamento Térmico ($\tau$)** do material, não pelo número de passos lógicos.
2. Analogia: A natureza minimiza a energia de uma gota d'água instantaneamente, usando o paralelismo massivo de $10^{23}$ moléculas.
3. O Kernel v3 utiliza esta física: a "computação" é realizada simultaneamente por todas as partes do sistema reagindo ao gradiente de energia.
